Chat tool vs Coding agent: đổi tư duy trước khi đổi cú pháp

Trước khi học cách viết prompt, cần thấy Claude Code khác công cụ bạn đang dùng ở đâu. ChatGPT/Copilot là trợ lý ngồi cạnh — bạn hỏi, nó trả lời, bạn copy. Claude Code là đồng nghiệp tự chạy trong máy bạn — đọc file, sửa file, chạy lệnh, ghi TODO. Hai mô hình khác nhau khiến cách viết prompt cũng phải khác để phù hợp hơn với luồng Agentic Workflow.

Core concept · 1/2

Chat tool vs coding agent: change your mindset before your syntax

Before learning how to write prompts, you need to see where Claude Code differs from the tools you use today. ChatGPT/Copilot is an assistant sitting beside you — you ask, it answers, you copy. Claude Code is a colleague that runs inside your machine — it reads files, edits files, runs commands, writes TODOs. The two models are different, so the way you write prompts has to differ too, to fit an agentic workflow.

Aspect CHAT TOOL
ChatGPT · Copilot		 CODING AGENT
Claude Code CLI
How you interactContext-free chat: you paste code and give a request, it spits code back. No project context.An agent running in your terminal that understands project context to read and write files directly.
Actual outputA code block in the chat — you have to copy it back.Edited files, commits created, tests run — the result lands right in your repo's files.
MemoryShort-term, not coding-specific.File system + persistent TODOs — quit and reopen, it's still there.
WorkflowEach answer is free-form, no structure.Has skills (reusable workflows) and sub-agents (planner, tester...) that divide the work.
Your roleThe one asking questions.The one directing, reviewing and deciding — the agent executes.
So: a prompt for Claude Code isn't a question — it's an instruction handing work to a skilled colleague who lacks context. Enough context → the agent is precise. Too little → the agent guesses and you clean up the mess.
Core concept · 2/2

The four building blocks behind every session

Skill

A reusable workflow package.

/morkit:brainstorming opens explore mode. /morkit:writing-plans creates a phased plan before coding.

Agent

A specialized alter-ego with its own context.

planner, tester, code-reviewer, debugger — Claude delegates the work.

Plan

A design doc written before you type code.

One overview plan.md + many detailed phase-XX-*.md.

TaskCreate

Persistent TODOs across sessions.

Quit the terminal and reopen — the TODO is still there. Not chat history.

Understand these four and every prompt you write has a clear purpose: trigger a skill, call an agent, create a plan, or update a task.
